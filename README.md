## 快速开始

### 1. 导入许可证

首次使用需要导入许可证文件（`.sbl` 格式）：

1. 打开应用，进入**设置**页面
2. 点击**导入许可证**按钮
3. 选择许可证文件

> ⚠️ 许可证文件创建后 **1 小时内** 必须导入，超时将无法使用

### 2. 配置机厅信息

在设置页面配置以下信息：

| 配置项    | 说明              | 示例             |
| ------ | --------------- | -------------- |
| 区域 ID  | 机厅所在区域编号        | `XX`           |
| 区域名称   | 区域名称            | `XX`           |
| 机厅 ID  | 机厅编号            | `XXXX`         |
| 机厅名称   | 机厅名称            | `XX机厅`         |
| 客户端 ID | 机台客户端ID（11位）    | `A63EXXXXXXXX` |

> 💡 使用线下常去的机厅信息，降低账号异常概率
> ⚠️ **客户端 ID** 为必填项，格式为 11 位字符，用于生成 keychipId（格式：前4位-中间7位+0000）

### 3. 开始使用

应用底部有两个标签页：

- **标签页「发送功能票」**：扫描二维码 → 自动登录 → 发送功能票 → 自动登出
- **标签页「获取账号数据」**：扫描二维码 → 自动登录 → 获取完整数据 → 配置自定义数据（显示原始值） → 构建上传 → 自动登出

选择对应标签页后：
1. 点击**扫描二维码**或**手动输入**
2. 扫描/输入公众号生成的二维码（支持自动读取剪贴板）
3. 等待自动处理完成（处理中显示进度面板和实时日志）
4. 查看日志确认结果（支持导出完整诊断日志）

## 功能说明

### 时间同步状态

主界面左上角（AppBar 左侧）显示时间同步状态图标：

| 图标 | 颜色    | 状态  | 说明                 |
| -- | ----- | --- | ------------------ |
| ✓  | 🟢 绿色 | 已同步 | 系统时间与网络时间偏差在 1 秒以内 |
| ⏱  | 🟠 橙色 | 检测中 | 正在通过 NTP 服务器检测时间   |
| ✗  | 🔴 红色 | 不同步 | 时间偏差超过 1 秒，点击查看详情  |

> 💡 时间同步检测使用 NTP 协议，并行查询所有服务器（aliyun/tencent/google），多轮中位数交叉验证确保准确性
> 💡 应用启动时自动检测，偏差超过容差时会弹出警告对话框

### 二维码处理

支持两种方式：

- **扫描二维码**：使用相机扫描二维码（需授予相机权限）
- **手动输入**：直接粘贴二维码文本内容（支持自动读取剪贴板）

二维码格式支持：
- `SGWCMAID` 前缀格式（长度 ≥ 84）
- 纯十六进制格式（长度 ≥ 64，`[0-9A-Fa-f]+`）

> 二维码有效期为 **10 分钟**，超时需重新获取
> 解析结果格式：12位时间戳 + 8位用户ID + token

### 功能票倍数

仅在「发送功能票」标签页可调整：

| 倍数 | 价格 | 安全性      |
| -- | -- | -------- |
| 2倍 | 1  | ✅ 安全     |
| 3倍 | 2  | ✅ 安全（默认） |
| 4倍 | 3  | ⚠️ 危险    |
| 5倍 | 4  | ⚠️ 危险    |
| 6倍 | 5  | ⚠️ 危险    |

> ⚠️ 4/5/6 倍功能票存在风险，请谨慎使用
> 💡 倍数通过 FilterChip 选择，选中后自动保存到机厅配置中

### 发票等待时间

固定20秒等待，无需配置。

> 💡 服务器速率限制要求，发票等待时间固定为 20 秒
> 💡 登录后先按顺序调用API获取数据和检查库存，再等待20秒后发票

### 许可证系统

#### 许可证类型

| 类型      | 说明             |
| ------- | --------------- |
| 单一用户许可证 | 仅限指定 UserID 使用 |
| 管理员许可证  | 不限制 UserID     |

#### 许可证有效期

- **永久有效**：无过期时间
- **限时有效**：可设置任意时长（最短1分钟）

#### 许可证安全机制

- **签名算法**：Ed25519 非对称签名
- **导入时效**：创建后 1 小时内必须导入
- **时间回拨检测**：容忍度 15 分钟，防止通过修改系统时间绕过验证
- **多密钥轮换**：通过 kid（密钥ID）支持公钥更换

#### 许可证验证时机

应用在以下关键操作节点会验证许可证：
1. **启动验证**：解析二维码成功后，检查许可证是否加载、有效、签名正确、用户ID授权
2. **登录前验证**：执行登录操作前的二次确认
3. **发票前验证**：发放功能票前的最终确认

### 自动更新

应用支持从 GitHub Release 检查更新：

1. 进入**设置**页面
2. 点击**检查更新**按钮
3. 如有新版本，点击**下载并安装**

> 💡 下载源：优先自建 IPv6 源，失败回退 GitHub Release
> 💡 版本检查使用 GitHub Releases API（`gczydg/SBGATOOLS_APP` 仓库 latest Release）
> 💡 应用会自动使用系统配置的网络代理（通过 MethodChannel 获取 Android 系统代理设置）
> 💡 下载完成后自动调用系统安装器打开 APK 文件

#### 自动检查更新

在设置页面"检查更新"下方可开启**自动检查更新**开关：

- 启用后，每次打开应用时会自动检查更新
- 若发现新版本，弹窗提示并提供"稍后提醒"和"前往更新"两个选项
- 默认不启用，需手动开启
- 开关状态持久化保存

### 更新公告

版本更新后首次打开应用时，会自动弹窗展示该版本的更新内容。确认后不再重复显示，直到下次版本更新。

### 获取账号数据

第二个标签页提供完整的存档修改功能。标签页顶部有"修改并上传"开关（默认开启）：

- **开启**：获取数据 → 暂停配置 → 构建上传 → 登出（完整存档修改流程）
- **关闭**：获取数据 → 直接登出（仅更新本地数据，不修改存档）

> 💡 开关状态仅内存有效，应用重启后恢复为默认开启

| GetUserChargeApi | 功能票库存 | rethrow 触发重试 |
| GetUserFavoriteApi | 收藏数据（5种 itemKind） | rethrow 触发重试 |
| GetUserMapApi | 地图进度数据 | rethrow 触发重试 |
| GetUserOptionApi | 游戏选项设置 | rethrow 触发重试 |
| GetUserExtendApi | 扩展数据 | rethrow 触发重试 |
| GetUserRatingApi | Rating 和段位数据 | rethrow 触发重试 |
| GetUserMusicApi | 音乐成绩数据 | rethrow 触发重试 |
| GetUserActivityApi | 活动数据 | rethrow 触发重试 |
| GetUserMissionDataApi | 任务数据 | rethrow 触发重试 |
| GetUserItemApi | 用户物品（礼物/歌曲解锁/收藏品，条件获取） | rethrow 触发重试 |
| GetUserKaleidxScopeApi | 万花筒区域数据（条件获取） | rethrow 触发重试 |
| GetUserRegionApi | 用户各省份游玩次数 | rethrow 触发重试 |

#### 可自定义字段

数据获取完成后自动弹出配置页面，每个配置项旁显示当前原始值，页面顶部提示"登录态保持中，请尽快完成配置"：

| 字段 | 说明 | 特殊行为 |
| ---- | ---- | -------- |
| `playerRating` | 临时显示 Rating | 上机游玩一把后自动还原为原始值 |
| `playCount` | 累计游玩次数 | UserAll 上传时 +1 |
| `currentPlayCount` | 当前版本游玩次数 | UserAll 上传时 +1 |
| `point`（舞里程） | 增加舞里程数量 | 同时增加 point 和 totalPoint |
| 增加礼物数量 | 购买礼物增加库存 | 原库存 + 增加数量，同时生成 userTradeItemList 购买记录 |
| 清票 | 发票后修改存档时清零功能票库存 | 自动（有票时启用，无票时禁用），显示倍数详情如"1张3倍票"，将未使用的功能票stock改为0 |
| 歌曲解锁（不含紫白谱） | 解锁需要特定条件的歌曲（仅歌曲本身，不含紫白谱难度） | Switch 启用/禁用，三种模式：手动/部分/全部 |

| 相同旅行伙伴 | 单推旅行伙伴开关 | 5个角色位置设为相同队长角色并锁定 |
| 旅行伙伴升级 | 旅行伙伴升级开关 | 将队内旅行伙伴修改至9999级觉醒6，显示所有装备角色的名称和等级 |
| 发放旅行伙伴 | 添加指定旅行伙伴到账号 | Switch 启用/禁用，选取窗口支持图片预览+名称/ID搜索+多选确认+持有状态筛选（3列布局，获取默认未持有），固定1级/未觉醒/0使用次数，已有伙伴自动跳过 |
| 友人对战 | 友人对战阶级和CP点数修改 | Switch 启用/禁用，DropdownButtonFormField 选择阶级，TextField 设置CP点数 |
| mapStock | 移动距离设为999KM | Switch 启用/禁用，启用后 mapStock=999000（999KM），默认关闭 |
| 收藏品 | 获取或移除头像/姓名框/背景/称号/搭档 | Switch 启用/禁用，Switch 移除模式，选取窗口（TabBar切换+搜索+多选+持有状态筛选，获取默认未持有/移除默认已持有；头像搭档3列/姓名框1列/背景1列/称号列表），自动查询已有收藏品避免重复 |
| 歌曲成绩上传 | 覆盖上传自定义歌曲成绩 | Switch 启用/禁用，至多3首歌曲，同时修改MusicDetailList和PlaylogList，包含通用信息和Playlog判定详情 |

> ⚠️ 数值不能为负数，负数输入会被忽略

#### 存档上传流程

构建 UserAll 数据后需要同时调用两个 API：
1. **GetUserNewItemListApi**：上传 UserLog（游玩记录摘要）
2. **UpsertUserAllApi**：上传完整 UserAll（用户所有数据）

两个 API 都返回成功才算上传完成。中间有动态等待期（lastLoginDate+61秒）模拟真实游玩间隔。

### 日志系统

#### 操作日志

- 实时显示每次操作的详细日志
- 支持四种级别：信息（蓝色）、成功（绿色）、警告（橙色）、错误（红色）
- 最多保留 100 条日志（新日志插入头部）
- 处理中切换到**进度面板**显示进度条和当前步骤

#### 日志导出

点击**导出**按钮可保存完整诊断日志，包含四部分：

| 部分 | 内容 |
| ---- | ---- |
| 第一部分 | 操作日志（带时间戳和级别） |
| 第二部分 | 网络通信详情（API名称/状态码/延迟/压缩率/Cookie变更） |
| 第三部分 | UserAll JSON（含 SHA256 校验和） |
| 第四部分 | UserLog JSON（含 SHA256 校验和） |

### 取消操作

处理过程中可点击**取消**按钮：
- 取消标志会在每个检查点被检测到
- 检测到取消后立即跳转到登出环节确保账号安全退出
- 处理中禁止滑动切换标签页和页面滚动

## 操作流程

### 发送功能票流程（6 步骤）

```
步骤1: 解析二维码 → 步骤2: 验证许可证+获取用户预览 → 步骤3: 等待1秒后登录
→ 步骤4: 获取数据+检查库存 → 步骤5: 等待20秒(固定)后发放功能票(如无票)
→ 步骤6: 等待1秒后登出并验证
```

### 获取账号数据流程（7 步骤）

**"修改并上传"开关开启时**：

```
步骤1: 解析二维码 → 步骤2: 验证许可证+获取用户预览 → 步骤3: 等待1秒后登录
→ 步骤4: 获取16类用户数据(每项重试3次) → 步骤5: 配置自定义数据(显示原始值，登录态保持中)
→ 步骤6: 构建UserAll数据 → 动态等待至lastLoginDate+61秒 → 上传GetUserNewItemListApi → 上传UpsertUserAllApi
→ 步骤7: 等待1秒后登出
```

**"修改并上传"开关关闭时**：

```
步骤1: 解析二维码 → 步骤2: 验证许可证+获取用户预览 → 步骤3: 等待1秒后登录
→ 步骤4: 获取16类用户数据(每项重试3次) → 步骤5: 等待1秒后登出
```

> 💡 步骤5中数据获取完成后自动弹出配置页，每个配置项旁显示当前原始值，页面顶部提示"登录态保持中，请尽快完成配置"。完成配置后继续自动执行后续步骤。

整个过程除步骤5需手动配置外自动完成，您只需：

1. 提供有效的二维码
2. 选择对应的标签页
3. 等待数据获取完成后，在弹出的配置页中调整自定义数据（配置项旁显示原始值）
4. 确认配置后等待处理完成（观察进度条和实时日志）
5. 查看日志确认结果（可选导出诊断日志）

## 常见问题

### Q: 提示"二维码已过期"

**A:** 二维码有效期为 10 分钟，请在公众号上获取新的二维码。

### Q: 提示"许可证不允许操作此用户ID"

**A:** 当前许可证未包含您的 UserID（单一用户许可证），请联系管理员获取授权或使用管理员许可证。

### Q: 提示"许可证导入时效已过"

**A:** 许可证文件创建后 1 小时内必须导入，超时请联系管理员补发新许可证。

### Q: 提示"许可证已过期"

**A:** 许可证已超过有效期，请联系管理员续期或重新获取。

### Q: 提示"检测到系统时间异常"

**A:** 系统检测到时间回拨（容忍度 15 分钟），可能是因为手动修改了系统时间。请重新导入许可证。

### Q: 发票失败

**A:** 可能原因：

- 账号内已存在功能票（库存 > 1 会导致数据失效，应用会跳过此步骤）
- IP 地址被服务器限制
- 网络连接问题
- 系统时间与标准时间偏差过大（建议开启"自动确定日期和时间"）
- 许可证验证未通过（管理员许可证不受影响）

### Q: 存档发送失败 / returnCode=0

**A:** 可能原因：

- 网络连接不稳定导致 API 调用失败（应用会自动重试最多 3 次，间隔 2 秒）
- IP 地址被封禁
- 系统时间偏差过大
- UserAll 数据构建失败（查看日志中的具体错误信息）

### Q: 扫描二维码不工作

**A:** 请在系统设置中授予应用相机权限。

### Q: 无法安装更新

**A:** Android 8.0+ 需要授予"安装未知应用"权限，应用会在下载完成后提示授权。

### Q: 检查更新/下载失败

**A:** 可能原因：

- 网络连接问题
- 如需使用代理，请在系统 WLAN 设置中配置 HTTP 代理
- GitHub 访问受限，请尝试使用代理或 VPN

### Q: 启动时弹出"时间同步警告"

**A:** 系统时间与网络时间偏差超过 1 秒（NTP 容差）。请：

- 前往系统设置开启"自动确定日期和时间"
- 或手动调整系统时间至正确时间
- 主界面左上角图标始终显示时间同步状态

### Q: "获取账号数据"中的自定义数据是什么意思？

**A:** 数据获取完成后会自动弹出配置页面，您可以临时覆盖用户的游玩数据。每个配置项旁会显示当前原始值，方便参考：

| 字段 | 效果 |
| ---- | ---- |
| `playerRating` | 临时显示指定 Rating，上机游玩一把后自动还原 |
| `playCount` | 修改累计总游玩次数 |
| `currentPlayCount` | 修改当前版本游玩次数 |
| 舞里程 | 增加舞里程数量，可在商店中用舞里程获取道具 |
| 增加礼物数量 | 购买礼物增加库存，通过送礼提升伙伴好感度 |
| 清票 | 根据功能票库存自动启用或禁用：有票时自动清零以避免上传失败，无票时保持禁用。显示倍数详情如"1张3倍票" |
| 歌曲解锁（不含紫白谱） | 解锁需要特定条件的歌曲（仅歌曲本身，不含紫白谱难度） |
| KaleidxScope解锁 | 解锁万花筒区域的门与钥匙，选择区域后自动发现门和获取钥匙。显示每个区域的门钥匙发现状态 |
| 相同旅行伙伴 | 将5个旅行伙伴设为相同的队长角色（单推模式） |
| 旅行伙伴升级 | 将队内旅行伙伴修改至9999级觉醒6。显示所有装备角色的名称和等级 |
| mapStock | 将移动距离设为999KM，可在地图中使用移动距离获取道具 |
| 收藏品 | 获取或移除头像/姓名框/背景/称号/搭档等收藏品，自动查询已有避免重复，移除模式设置stock=0，窗口选取（TabBar切换+搜索+多选+持有状态筛选，获取默认未持有/移除默认已持有） |
| 友人对战 | 修改友人对战阶级和CP点数，选择阶级后自动计算对应CP值，显示当前段位和CP |
| 歌曲成绩上传 | 覆盖上传自定义歌曲成绩，至多3首，同时修改MusicDetailList和PlaylogList |

> 配置1次执行内有效，执行完成后自动清除

### Q: "修改并上传"开关有什么作用？

**A:** 该开关位于"获取账号数据"标签页顶部，默认开启：

- **开启**：获取数据后暂停流程，弹出配置页供您修改自定义数据，确认后构建上传存档并登出
- **关闭**：获取数据后直接登出，仅更新本地数据，不修改服务器存档

> 💡 开关状态仅内存有效，应用重启后恢复为默认开启

### Q: 日志导出文件在哪里？

**A:** 日志导出时会弹出文件选择对话框，您可以选择保存位置。导出文件为 `.txt` 格式，包含完整的操作日志和网络诊断信息。

### Q: 处理过程中可以取消吗？

**A:** 可以。处理过程中**执行**按钮变为**取消**按钮，点击后：
- 当前操作不会立即中断，而是在下一个安全检查点跳转到登出环节
- 确保账号始终能安全退出
- 取消后日志会记录"正在取消操作，将进入登出环节"

## 注意事项

1. **二维码时效**：有效期 10 分钟，请及时使用
2. **许可证时效**：创建后 1 小时内必须导入
3. **功能票倍数**：高倍数（4/5/6 倍）存在风险，建议使用默认的 3 倍
4. **发票等待时间**：固定 20 秒等待
5. **网络环境**：确保网络连接稳定，如需代理请在系统 WLAN 设置中配置
6. **机厅配置**：使用真实的机厅信息可降低账号异常风险
7. **系统代理**：应用自动读取 Android 系统 HTTP 代理配置
8. **时间同步**：启动时自动 NTP 检测，偏差超过 1 秒时请及时同步
9. **beta 功能**："获取账号数据"功能请谨慎使用，数据获取后自动弹出配置页（显示原始值），配置时登录态保持中，请尽快完成配置；关闭"修改并上传"开关可仅获取数据不修改存档
10. **安全登出**：无论操作成功或失败，应用都会尝试安全登出账号
11. **重试机制**：数据获取阶段每项 API 失败会自动重试 3 次（间隔 2 秒）
12. **日志保留**：操作日志最多 100 条，网络诊断日志最多 500 条
