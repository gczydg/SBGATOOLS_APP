# SBGATOOLS 使用手册

> **当前版本**: v1.4.14+111
> 一款用于 maimai DX 游戏的功能票发放与存档修改工具。

## 快速开始

### 1. 导入许可证

首次使用需要导入许可证文件（`.sbl` 格式）：

1. 打开应用，进入**设置**页面
2. 点击**导入许可证**按钮
3. 选择许可证文件

> ⚠️ 许可证文件创建后 **1 小时内** 必须导入，超时将无法使用

### 2. 配置机台狗号

在设置页面填写机台狗号（clientId）。区域、店铺 ID、店铺名称和认证时间由 APP 执行 ALL.Net PowerOn 后自动获取并缓存，设置页只读显示当前获取到的信息。

> ⚠️ 机台狗号为必填项，格式为 11 位字符；如果 PowerOn 返回狗号失效，APP 会阻止后续 Title 通信。

### 3. 开始使用

应用底部有两个标签页：

- **发送功能票**：扫描二维码 → 自动登录 → 发送功能票 → 自动登出
- **获取账号数据**：扫描二维码 → 自动登录 → 获取完整数据 → 配置自定义数据 → 上传存档 → 自动登出

操作步骤：
1. 点击**扫描二维码**或**手动输入**
2. 扫描/输入公众号生成的二维码（支持自动读取剪贴板）
3. 等待自动处理完成（处理中显示进度和实时日志）
4. 查看日志确认结果（支持导出诊断日志）

## 功能说明

### 时间同步与 PowerOn 状态

主界面左上角显示 NTP 时间同步和 PowerOn 的合并状态：

| 图标 | 颜色 | 说明 |
| -- | -- | -- |
| ✓ | 🟢 绿色 | NTP 同步成功，PowerOn 认证成功或已命中当日缓存 |
| ⏱ | 🟠 橙色 | 正在进行 NTP/PowerOn 检查，或 PowerOn 使用当日缓存 |
| ✗ | 🔴 红色 | NTP 不可用、PowerOn 失败，或狗号失效 |

> 💡 应用启动时自动进行 NTP 同步和 PowerOn。NTP 会解析服务器原始时间响应并过滤高延迟/异常样本；PowerOn 会获取 `utc_time`、`client_timezone`、区域和店铺信息，并派生 Title 通信用的 `dateTime`。同步或 PowerOn 检查期间不能点击执行或扫描二维码。

### 二维码处理

支持**扫描二维码**（需相机权限）和**手动输入**（支持自动读取剪贴板）两种方式。

> 二维码有效期为 **10 分钟**，超时需重新获取

输入二维码后，输入框下方会显示剩余有效期倒计时，颜色随剩余时间变化：

| 颜色 | 含义 | 可提交 |
| -- | -- | -- |
| 🟢 绿色 | 剩余 ≥1 分钟，正常有效 | ✓ |
| 🟠 橙色 | 剩余 <1 分钟，临近过期 | ✓（请尽快提交） |
| 🔴 红色 | 已过期 | ✗ |
| ⚪ 灰色 | 格式无效或未输入 | ✗ |

> 💡 临近过期时倒计时变橙色提示，仍可提交但请尽快操作；过期后提交会被拦截并提示重新获取

### 功能票倍数

仅在「发送功能票」标签页可调整：

| 倍数 | 价格 | 安全性      |
| -- | -- | -------- |
| 2倍 | 1  | ✅ 安全     |
| 3倍 | 2  | ✅ 安全（默认） |
| 5倍 | 4  | ⚠️ 危险    |

> ⚠️ 5 倍功能票存在风险，请谨慎使用
> ⚠️ 4 倍和 6 倍功能票因游戏版本 1.55 起不再支持，已移除

### 发票等待时间

固定 20 秒等待，无需配置。计时从 **UserLoginApi 响应成功返回** 后开始，获取用户数据和库存检查会在等待窗口内完成，上传前只补足剩余等待时间。

执行日志中等待阶段统一显示为 `【步骤5】发票处理（等待x秒）...`；账号已有功能票并跳过发票时不显示等待日志。

### 许可证系统

- **单一用户许可证**：仅限指定 UserID 使用
- **管理员许可证**：不限制 UserID
- 支持永久有效和限时有效两种有效期
- 创建后 1 小时内必须导入
- 执行时会在二维码解析后验证许可证、用户权限和可信网络时间新鲜度

### 自动更新

进入**设置**页面点击**检查更新**，也可开启**自动检查更新**（每次打开应用时自动检测新版本）。

更新检查、APK 下载和远程公告均使用设备系统信任的 TLS 证书链；证书无效或未受信任的 HTTPS 拦截会被拒绝。自建下载源证书异常时，已获取下载地址的更新会自动回退至 GitHub。

自建下载源仅支持 IPv6。下载前会检测当前 IPv6 连通性；无 IPv6 时会跳过自建源并直接使用 GitHub，避免无效等待。

### 更新公告

版本更新后首次打开应用时，会自动弹窗展示更新内容。跨多版本升级时，弹窗会累积展示从上次查看版本之后到当前版本的全部更新内容，不再遗漏中间版本。

### 获取账号数据

第二个标签页提供完整的存档修改功能。标签页顶部有"修改并上传"开关（默认开启）：

- **开启**：获取数据 → 配置修改 → 上传存档 → 登出
- **关闭**：获取数据 → 直接登出（仅更新本地数据，不修改存档）

存档上传流程已按 SDGB 1.55 DLL 等价逻辑迁移：登录后构建 `UserLog` 与 `UserAll`，先上传 `GetUserNewItemListApi`，再上传 `UpsertUserAllApi`。上传存档的 60 秒等待同样从 UserLoginApi 响应成功返回后开始计时；等待结束前会刷新实际上传时刻相关字段。执行日志先显示 `【步骤6】上传数据（等待x秒）...`，随后依次显示 `上传 GetUserNewItemListApi...` 和 `上传 UpsertUserAllApi...`。`playSpecial` 随机值按 DLL/.NET 的 signed int32 语义发送，避免高位随机值被错误当作 unsigned 正数导致服务器 HTTP 500。

**本地数据同步**：存档上传成功（`returnCode=1`）后，APP 会自动用最终 UserAll 数据增量更新本地数据库至上传后的最新状态。此前数据页展示的是上一次获取时的状态（会落后一次），现上传后立即同步——userData 全量替换，角色/收藏品/地图/集章卡/歌曲成绩按主键增量更新（收藏品移除模式会删除对应记录）。同步失败不影响主流程，仅记录警告日志。

### 数据页面

第三个标签页提供账号数据浏览和收藏品/歌曲查询：

- **账号页**：展示用户信息、DX Rating、游玩统计、当前装备，账号预览窗和游玩统计窗之间提供查分器同步入口（"水鱼同步"将本地歌曲成绩上传至水鱼查分器；"落雪同步"将本地歌曲成绩上传至落雪咖啡屋查分器，原生支持宴会场曲目）
- **收藏品查询**：按种类浏览头像、姓名框、背景、称号、搭档的ID、名称和图片，支持按持有状态筛选；点击已拥有的头像/旅行伙伴可查看详情（旅行伙伴显示当前等级，超觉醒格式如"10级1星超觉醒"，普通觉醒格式如"998级觉醒4"）
- **歌曲查询**：按分类（标准/DX/宴会会场）和版本筛选歌曲，查看歌曲详情（含谱面物量、理论DX分和游玩成绩DX分）；版本筛选按当前 musicId 对应谱面的收录版本分类，标准谱、DX 谱和宴会场谱互不混用；所有版本页统一按大版本号倒序分组排列，同大版本内普通谱在前（按版本号倒序）、宴会场垫底（按版本号倒序），同版本号内按 musicId 正序

> 同一首歌的标准谱和 DX 谱可能在不同版本收录。例如歌曲 1103 的标准谱归入舞萌DX 2024，而 DX 谱 11103 仍归入舞萌DX 2021；1301/11301 也是同类情况。筛选器会分别显示在正确的版本中。

> v1.4.13 起歌曲排序由按 musicId 倒序改为按版本号倒序（同版本号内按 musicId 正序），解决国服旧歌同步到新版本时因 musicId 较大始终排在顶部的问题。数据页宴会场按大版本分组后垫底（因 LXNS 宴会场版本号与普通谱存在不一致），歌曲传分选择器完全不显示宴会场。

> v1.4.9 起已修复账号页顶部 Rating 数字预览的黑色虚影问题：原渲染结构中 `FractionallySizedBox` 在 `Stack(fit: StackFit.expand)` 内的定位语义不稳定，导致数字叠加层位置错乱形成虚影；现已改用 `LayoutBuilder + Positioned` 精确定位，并移除数字阴影。

### 日志系统

- 实时显示操作日志，支持四种级别（信息/成功/警告/错误）
- 流程进度日志统一为"流程名称（等待x秒）"格式；上传结果只保留包含处理建议的详细日志，避免同一结果重复显示
- 区域进度、集章卡、收藏品和歌曲解锁等变更日志会尽量显示具体名称与 ID，区域奖励会区分获取项和跳过已有项
- 点击**导出**按钮可保存完整诊断日志（含操作日志、maimai 服务器通信详情、实际发生的 LXNS 歌曲 API 请求及失败原因、上传数据校验）

### 取消操作

处理过程中可点击**取消**按钮，应用会在下一个安全检查点跳转到登出环节，确保账号安全退出。

### 后台防杀

执行期间自动启动前台服务，切换到后台或小窗状态不会中断流程。状态栏显示常驻通知，通知上的"取消"按钮可随时取消操作。

### 异常退出自动登出

如果应用在执行过程中意外关闭（如系统杀进程），下次检测到该账号已登录时，会自动使用保存的会话信息尝试登出，确保账号安全。

## 常见问题

### Q: 提示"二维码已过期"

**A:** 二维码有效期为 10 分钟，请在公众号上获取新的二维码。

### Q: 提示"许可证不允许操作此用户ID"

**A:** 当前许可证未包含您的 UserID，请联系管理员获取授权或使用管理员许可证。

### Q: 提示"许可证导入时效已过"

**A:** 许可证文件创建后 1 小时内必须导入，超时请联系管理员补发新许可证。

### Q: 提示"许可证已过期"

**A:** 许可证已超过有效期，请联系管理员续期或重新获取。

### Q: 提示"可信网络时间不可用"

**A:** NTP 同步失败或可信网络时间已过期。请检查网络后重新执行；同步期间应用会禁止开始执行。

### Q: 提示"PowerOn 狗号失效"或"PowerOn 认证失败"

**A:** 狗号无效、过期，或当前网络无法完成 ALL.Net PowerOn。请确认设置页填写的是有效 11 位狗号；如果只是网络临时失败，APP 会优先使用当天 07:00 游戏日内的成功缓存，没有可用缓存时会阻止执行。

### Q: 发票失败

**A:** 可能原因：

- 账号内已存在功能票（应用会自动跳过）
- IP 地址被服务器限制
- 网络连接问题

### Q: 存档发送失败

**A:** 可能原因：

- 网络连接不稳定（应用会自动重试最多 3 次）
- IP 地址被封禁

> v1.4.5 起已修复一类旧版本随机失败：旧逻辑在 `playSpecial` 随机到高位 32-bit 值时会以 unsigned 正数上传，可能触发 `UpsertUserAllApi` HTTP 500。该问题与系统时间同步无关，表现为“有时成功、有时失败”。

### Q: 扫描二维码不工作

**A:** 请在系统设置中授予应用相机权限。

### Q: 无法安装更新

**A:** Android 8.0+ 需要授予"安装未知应用"权限，应用会在下载完成后提示授权。

### Q: 检查更新/下载失败

**A:** 可能是网络问题或 GitHub 访问受限，请尝试使用代理或 VPN。

### Q: 歌曲成绩上传的"Playlog 真实化"是什么？

**A:** 开启后会从 LXNS 获取谱面物量，并按 DLL 计分规则生成 playlog。手动输入 achievement 时优先保证 `comboStatus` 和 `scoreRank` 正确，achievement 可能调整到最接近的真实值；手动修改 25 个判定值时会反向正算出精确 achievement。AP/AP+ 边界按游戏实际规则处理：所有谱面都有 BREAK，AP 必须包含至少 1 个 Break Perfect，1010000 且全 Critical 会变成 AP+。若账号 `dispJudge` 为 Type1 系列，普通键 Critical 会合并显示到 Perfect，BREAK 的 Critical/Perfect 仍保持分开。

v1.4.14 起，配置页成功取得并校验谱面物量后，最终构建会复用同一份物量，不再为同一谱面强制发起第二次 LXNS 请求。若物量仍在加载、缺失、校验失败或与当前歌曲/难度不匹配，保存会留在配置页并提示；从未取得有效物量时，真实化仍会阻止构建。导出的完整诊断日志会记录实际发生的 LXNS 请求及失败原因，命中已验证物量快照时不会伪造或额外发起请求。

v1.4.5 起，删除歌曲行后实时计算缓存会同步重排，避免后续歌曲继承错误的真实 achievement/combo；LXNS 返回全零物量会直接阻止真实化构建。关闭"Playlog 真实化"时仍走旧娱乐模板，这是兼容路径，不应当作死代码删除。

v1.4.13 起歌曲传分禁止重复谱面：同一列表内不允许选择同一首歌的同一难度。难度下拉会自动灰显并禁用其他行已选的谱面，保存时也会兜底校验。原因是每份 playlog 都对应一条 `userMusicDetailList` 记录，服务器按 (musicId, level) 唯一约束，重复谱面会导致 `UpsertUserAllApi` 假 returnCode=1 丢弃整次上传。

v1.4.6 起补充了拆分区块的父页面刷新兜底，避免自定义数据页开关只更新服务值、不立即更新显示状态；同时将 PowerOn 当日缓存日切改为 07:00，与账号每日刷新和服务器维护结束口径保持一致。

v1.4.8 起修复了修改判定值后 deluxscore 输入框和 comboStatus 下拉未同步更新真实计算值的问题——现在修改任意一项（achievement/comboStatus/deluxscore/判定值）均会触发实时计算并把真实值回填到所有相关 UI 控件，确保界面显示与实际提交数据一致。同时修正了 AP+/AP 边界判定注释：AP+ 要求 BREAK 全 CP（BREAK 的 CP/P 通过 bonusScore 影响总 achievement，P75/P50 的 bonusLoss>0 会导致降分），AP 要求 BREAK 有 P（P75/P50）使 achievement<1010000；普通 note 的 CP/P 不影响 achievement，因此不影响 AP/AP+ 派生，只影响 deluxscore（CP=3, P=2）。计分表经 SDGB 1.55 DLL 反编译验证完全正确，TOUCH 与 TAP 计分同等（Wiki 表 TOUCH GREAT=1×评价値 为错误描述）。

v1.4.8 起修复了 AP+ 状态下普通 note 的 Perfect 判定档位不可编辑的问题。根据萌娘百科记载的真实语义，AP+（ALL PERFECT+）只要求 BREAK 类 Note 全部以 CRITICAL PERFECT 完成，普通 note 的 CP/P 不影响 achievement 与 combo 派生，应允许编辑。修复后 AP+ 下普通 note（Tap/Hold/Slide/Touch）的 P 档可编辑，BREAK 的 P75/P50 仍保持不可编辑（会降 achievement 导致 combo 降级为 AP）；同时支持 AP+ 下通过普通 note CP→P 调整 deluxscore，求解器会自动匹配用户输入的 targetDeluxscore。各 comboStatus 的可编辑档位对照：AP+ 普通 note CP/P + BREAK 仅 CP；AP 所有 note CP/P；FC+ 增加 GR；FC 增加 GD；none 增加 MS。

v1.4.8 起修复了反向求解触发时机问题。判定值已被手动编辑后，修改 achievement/comboStatus/deluxscore 不再触发反向求解（避免覆盖用户判定输入）；deluxscore 输入框改为失去焦点时才触发求解器，输入过程中不会被回填覆盖；playlog 正向计算保留物量异常警告（物量不一致时红色提示，物量正确时才触发回填）。

v1.4.8 起歌曲传分 UI 新增 DX 分数评级（✦星级）预览显示，与达成率评级（SSS+/SSS/...）并排显示。修改 deluxscore 或判定值后，DX 分数评级实时更新，直观展示当前 deluxscore 占比对应的星级（- / ✦ / ✦✦ / ✦✦✦ / ✦✦✦✦ / ✦✦✦✦✦，阈值 85%/90%/93%/95%/97%），与游戏内 DX 分数评级一致。

### Q: 水鱼查分器同步是什么？如何使用？

**A:** 数据页账号子页在账号预览窗和游玩统计窗之间提供"查分器同步"卡片，可将本地保存的账号歌曲成绩上传至水鱼查分器（diving-fish maimaidxprober），用于记录游玩成绩、输出达成进度和推分建议等。

- **首次使用**：点击"水鱼同步"按钮，弹出对话框引导输入 Import-Token（在水鱼查分器网站 https://www.diving-fish.com/maimaidx/prober/ 登录后，于"编辑个人资料"中生成）。Token 保存在本地数据库中与 UserID 对应，多账号互不影响。
- **后续同步**：绑定 Token 后再次点击会弹出确认对话框，显示将上传的记录数，提供"取消"、"更换Token"、"立即同步"三个选项；确认同步后按水鱼官方 API 要求转换本地成绩（达成率/10000、ComboStatus/SyncStatus 枚举映射、SD/DX 类型判断）并上传到 `POST /api/maimaidxprober/player/update_records`。
- **更换Token**：在同步确认对话框中点击"更换Token"按钮，会重新弹出 Token 输入对话框。
- **解除绑定**：在设置页删除对应账号数据，会同步删除该账号的水鱼 Token 绑定记录。
- **过滤规则**：宴会场歌曲（musicId≥100000）水鱼不支持，自动跳过；本地未知名歌曲因 title 无法确定也跳过。musicId 11272（"約束"）不过滤——它本身是真实歌曲，项目内部用其 level 0 作为虚拟占位 playlog 不影响查分器同步的真实成绩。
- **特殊处理**：musicId 383（"Link(CoF)"）的 title 上传为 "Link(CoF)"，与 musicId 131（"Link"）区分，避免水鱼数据库识别错误。
- **Token 失效**：服务器返回 400 + "导入token有误"时自动清除本地绑定，需重新输入 Token。
- **Token 输入**：输入框支持剪贴板粘贴（Token 极长且不规律，必须支持复制粘贴），不启用自动纠正和建议以避免某些输入法误判为安全键盘模式。

### Q: 落雪查分器同步是什么？如何使用？

**A:** 数据页查分器同步卡片的"落雪同步"按钮，可将本地保存的账号歌曲成绩上传至落雪咖啡屋查分器（maimai.lxns.net），用于记录游玩成绩和查看达成进度。

- **首次使用**：点击"落雪同步"按钮，弹出对话框引导输入个人 API 密钥（在落雪咖啡屋 https://maimai.lxns.net/ 登录后，于"账号详情"中生成）。密钥保存在本地数据库中与 UserID 对应，多账号互不影响。
- **后续同步**：绑定密钥后再次点击会弹出确认对话框，显示将上传的记录数，提供"取消"、"更换Token"、"立即同步"三个选项；确认同步后按落雪官方 API 要求转换本地成绩并上传到 `POST /api/v0/user/maimai/player/scores`。
- **与水鱼同步的差异**：落雪原生支持宴会场曲目（type=utage）成绩上传，标准与 DX 谱面共享同一曲目 ID（DX 谱面 musicId 会取模 10000 转换）；请求头使用 `X-User-Token`，请求体为 `{"scores":[...]}` 对象包裹数组。
- **更换Token**：在同步确认对话框中点击"更换Token"按钮，会重新弹出密钥输入对话框。
- **解除绑定**：在设置页删除对应账号数据，会同步删除该账号的落雪密钥绑定记录。
- **Token 失效**：服务器返回 401 时自动清除本地绑定，需重新输入密钥。
- **Token 输入**：输入框支持剪贴板粘贴（密钥较长且不规律，必须支持复制粘贴），不启用自动纠正和建议以避免某些输入法误判为安全键盘模式。

### Q: 数据页歌曲详情的"收录版本"对部分新歌显示为最新版本？

**A:** 数据页歌曲子页的“收录版本”实时读取 LXNS，列表的版本筛选读取生成的静态对照表；两条链路都按当前 musicId 对应谱面类型判断：标准谱取 standard、DX 谱取 dx、宴会场取 utage，不会使用另一种谱面的版本。即使同一 song_id 的谱面在不同版本收录，也会分别归类；例如 1103 的 standard 谱为舞萌DX 2024，11103 的 DX 谱仍为舞萌DX 2021，1301/11301 也是同类情况。游戏内 DX 谱面的 musicId（如新版 Dragoon 的 10367）会通过取模转换为 song_id（367）请求 LXNS；若 LXNS 中该 song_id 存在但对应类型谱面为空（367 的 DX 谱面未收录，只有旧版 standard 谱面），详情页“收录版本”会回退显示为当前最新版本号，而非错误显示同名旧曲的版本。若 LXNS 完全未收录该 song_id（如 11847），则详情页无法打开。标题/艺术家/流派/BPM 仍取自同名旧曲数据——新旧版是同一首歌的不同谱面版本，这些字段相同。

### Q: 系统显示大小设置过大时，歌曲详情的表格会显示异常吗？

**A:** 不会。数据页歌曲详情的物量表和游玩成绩表已适配系统显示大小设置——内容窄时表格撑满可用宽度不留空，内容宽（如调大系统字体后）时自动支持横向滚动，达成率、评级等列内容完整显示不会被挤压。对任意显示缩放比例（100%/125%/150%/200%…）都成立。

### Q: 收藏品列表中的"从整体中随机选择"/"从收藏中随机选择"是什么？

**A:** 这是游戏内"随机"功能的两种特殊设置状态（ID=2/3），不是实际存在的收藏品。数据页收藏品子页和获取收藏品选择器只显示实际存在的收藏品，不显示这两项；但设置收藏品选择器例外，允许用户选择随机装备模式作为装备值；对照表保留它们的 ID 映射，当用户装备了这两个随机状态时（游玩时可选择），账号子页装备预览和已选收藏品列表仍能正确显示名称。

### Q: "修改并上传"开关有什么作用？

**A:** 该开关位于"获取账号数据"标签页顶部，默认开启：

- **开启**：获取数据后弹出配置页供您修改，确认后上传存档
- **关闭**：获取数据后直接登出，仅更新本地数据，不修改服务器存档

### Q: 日志导出文件在哪里？

**A:** 日志导出时会弹出文件选择对话框，您可以选择保存位置。导出文件为 `.txt` 格式。

### Q: 处理过程中可以取消吗？

**A:** 可以。点击**取消**按钮后，应用会在下一个安全检查点跳转到登出环节，确保账号安全退出。

## 注意事项

1. **二维码时效**：有效期 10 分钟，请及时使用
2. **许可证时效**：创建后 1 小时内必须导入
3. **功能票倍数**：5 倍存在风险，建议使用默认的 3 倍（4 倍和 6 倍因游戏 1.55 版本起不再支持，已移除）
4. **发票等待**：固定 20 秒等待，从登录响应成功返回后计时
5. **网络环境**：确保网络连接稳定，如需代理请在系统 WLAN 设置中配置
6. **机厅配置**：仅手动填写机台狗号；区域、店铺和 Title `dateTime` 来自 PowerOn，店铺 ID 会从 PowerOn 十六进制 `place_id` 转换为十进制
7. **时间同步与 PowerOn**：启动时自动 NTP 检测和 PowerOn；执行前必须拥有新鲜可信网络时间和可用 PowerOn 状态，同步/认证期间不能开始执行
8. **存档修改**：请谨慎使用，配置时登录态保持中，请尽快完成配置；保存后若未到上传边界会继续等待，到时自动上传
9. **安全登出**：无论操作成功或失败，应用都会尝试安全登出；异常退出后下次检测到已登录会自动尝试登出
